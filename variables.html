Every variable.
Every default. Every format.

The technical reference for everything that gets templatized when this system clones from Waterloo HQ to a new franchise. Eyad uses this when building in Klaviyo. Engineering uses this when wiring data sources. This is the spec that prevents "Hey ," emails and franchise-cloning disasters.

The fallback rule

Every {{variable}} reference in actual Klaviyo templates must use Klaviyo's default-value syntax. The mockups show {{first_name}} for readability, but the production templates require fallbacks.

Mockup vs. production template

In the mockups: Hey {{first_name}},

In the actual Klaviyo template: Hey {{ first_name|default:'there' }},

Without the fallback, profiles signed up by phone, social, or any flow without a name will receive emails opening with "Hey ,". This is the single most-multiplied error if missed: every profile without a name × every email × every flow × every franchise location.

Click-property wiring spec

Three flows use Klaviyo's update_property_link mechanic — clicks set profile properties, which trigger downstream segmentation and flows. If any of these are wired wrong, real customers get suppressed. Test all 3 before launch.

F01.T05 + F08.T02 · 4-camp self-segment

Each "[click]" link in the 4-quadrant grid is wired to set player_type:

{% update_property_link 'player_type' 'solo' 'https://therunrec.com/thank-you' %}

{% update_property_link 'player_type' 'crew' 'https://therunrec.com/thank-you' %}

{% update_property_link 'player_type' 'party' 'https://therunrec.com/thank-you' %}

{% update_property_link 'player_type' 'league' 'https://therunrec.com/thank-you' %}

Test path: open the email in preview → click each link → verify the player_type property is set on the test profile. If the property isn't writing, the click is doing nothing and the segmentation breaks.

F13.T01 · sunset permission

"Click YES" link wires to engagement_intent:

{% update_property_link 'engagement_intent' 'stay' 'https://therunrec.com/thanks-for-staying' %}

The "monthly only" PS link:

{% update_property_link 'email_frequency' 'monthly' 'https://therunrec.com/preferences-saved' %}

The "unsubscribe" link is Klaviyo's standard {% unsubscribe %} token.

Critical: the auto-suppression rule (Day 14: anyone who hasn't clicked stays gets suppressed) must reference engagement_intent != 'stay' AND email_frequency != 'monthly'. If you suppress anyone with monthly intent, you delete real customers.
